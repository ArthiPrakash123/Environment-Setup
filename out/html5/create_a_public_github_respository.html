Creating a Public GitHub Respository

Create a public GitHub repository to serve a common location for your audience to view and access your DITA XML source files and published outputs.

Sign in to your personal GitHub account. For more information, see: Signing in to GitHub.

Use this procedure to create a public GitHub repository.

  1. On an instance of a web browser, type www.github.com.
  2. Sign in using your credentials.
  3. On the top right corner of the page, click the drop-down arrow next to the plus (+) sign, and click New repository.
    The Create a new repository page displays. Note that the user ID with which you sign in to GitHub appears as the Owner.
  4. Create a GitHub repository.
    1. In the Repository name* field, type a name for your repository of project files. This field is mandatory.
      For example, type: DITA-XML sample.
    2. Optional: Enter a description for the repository.
    3. Select Public.
      Important: Ensure that your repository is a public one. That way anyone who has the URL to the repository can access it.
    4. Click Create repository.
